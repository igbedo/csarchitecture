Evaluation Domains

Fellows are evaluated across five domains that collectively define modern cloud and security architecture. These domains make the program scalable and ensure the designation remains meaningful.

Evaluation is rubric-based. Associate instructors can support grading, while final designation is approved through defense review.

  1. Architecture & Systems Design — failure modeling, tradeoffs, clarity
  2. Infrastructure Automation — IaC structure, CI/CD promotion, reproducibility
  3. Cloud Security Architecture — IAM, segmentation, telemetry, threat surface
  4. Detection & Response Engineering — signal quality, workflows, response readiness
  5. AI-Augmented Operations — AI-assisted workflows with validation and guardrails

ACSA-F Designation

Autonomous Cloud & Security Architect – Fellow

ACSA-F is a post-nominal designation awarded only to Fellows who successfully complete the capstone and defend their architecture before review. This is an earned credential — not a participation certificate.

How ACSA-F is earned

  • Complete all required labs and submissions
  • Achieve performance thresholds across domains
  • Only top performers (≈80%) qualify for defense eligibility
  • Deliver capstone architecture build + documentation
  • Pass live architecture defense under questioning

Fellows who complete the program but do not earn ACSA-F receive a certificate of completion.

Admissions

Admission is selective. This fellowship is intended for professionals ready for architectural depth.

Who should apply

Cloud/DevOps/SRE professionals, platform engineers, and security engineers transitioning to architecture. Typically 2+ years in infrastructure roles.

How selection works

Written application → technical baseline assessment → interview (shortlist) → acceptance + required pre-work.

Pre-work required

Accepted Fellows complete an automation + architecture baseline prior to Week 1. Seats are forfeited if pre-work is not completed.

Not for beginners

This is not entry-level training, passive learning, or tool tutorials. Apply only if you can commit 10–15 hours per week and are prepared to be evaluated.
